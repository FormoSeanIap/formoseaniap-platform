# Articles Build Pipeline (Python)

## Summary

The article system is static at runtime and generated at build time.

- Source: `content/articles/**` (frontmatter-based and legacy nested markdown supported)
- Build tool: `scripts/build_articles.py`
- Output: `site/data/**` JSON + `site/rss.xml` + `site/zh/rss.xml`
- Serving: static hosting from `site/` (S3/CloudFront compatible)

## Why This Approach

- No backend/API/database required
- Content remains author-friendly (Markdown)
- Frontend consumes lightweight metadata index and per-article payloads
- RSS can be generated in the same build step

## Output Files

Generated by `python3 scripts/build_articles.py`:

- `site/data/articles.index.json`
  - metadata-only listing for filters/list page
- `site/data/articles/en/<id>.json`
- `site/data/articles/zh/<id>.json`
  - full article payload (metadata + `body_html`)
- `site/assets/articles/**`
  - copied article assets (images) from `content/articles/**`
- `site/rss.xml`
- `site/zh/rss.xml`

## Frontend Pages

- `site/articles.html`
  - loads `site/data/articles.index.json`
  - supports language/category/tag filtering
  - supports series filter via `?series=<series_id>&lang=<lang>`
- `site/article.html?id=<id>&lang=<lang>`
  - loads `site/data/articles/<lang>/<id>.json`
  - supports series navigation (`series_previous`, `series_next`) and "Series home" jump

## Author Workflow

1. Write article Markdown in `content/articles/<id>/en.md` and/or `zh.md`
2. Run build:
   - `python3 scripts/build_articles.py`
3. Verify in browser with static server
4. Deploy `site/` directory to S3

Notes:
- If frontmatter is missing, the builder infers metadata from path and content.
- For multi-part content (`Part1`, `Part2`, ...), previous/next links are auto-generated.
- If `external_url` is set in frontmatter, list/detail pages expose external-link mode.
- Image captions must use explicit image titles on image-only lines: `![alt text](./image.png "Visible caption")`.
- Inline images inside paragraphs do not render visible captions.
- Optional one-time migration to inject inferred frontmatter into legacy markdown:
  - `python3 scripts/normalize_frontmatter.py`
